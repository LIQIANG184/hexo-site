##项目管理
项目管理分为两部分：服务端项目管理，本地拍摄数据管理

拍摄流程一般是：
>创建本地项目 -> 进行拍摄 -> 拍摄完成点击生成vr -> 创建服务端项目 -> 关联本地项目 -> 项目上传 -> 服务端处理 -> 处理完成

###服务端项目管理(api)
服务端项目管理，是通过我们开放平台接口进行操作的，下面会介绍接口的参数和返回，demo中对所有接口进行了封装，可进行参考实现
`AppDelegate+CaptureSDK`

我们开放平台的 host 为：
`http://app-gateway.realsee.com`

####获取token
demo中参考
`AppDelegate+CaptureSDK`和token缓存机制`BSToken`

```
RSConfig.shareConfig.ticket = ^(RSCaptureTicketBlock  _Nonnull block) {
        //先判断是否存在有效token
        if (BSToken.share.currentToken != nil) {
            if(block){
                block (BSToken.share.currentToken);
            }
            return;
        }
        
        NSDictionary *dict = @{@"app_key":BSAPI.appKey,
                               @"app_secret":BSAPI.appSecret
        };
        [BSNetworkManager sendRequestWithMethod:BSRequestMethodTypePost path:BSAPI.accessToken parameters:dict config:nil completionBlock:^(id  _Nullable responseObject, NSError * _Nullable error) {
            if(error){
                block(nil);
            }else{
                NSDictionary *dict = responseObject[@"data"];
                NSString *access_token = dict[@"access_token"];
                BSToken.share.currentToken = access_token;
                if(block){
                    block (access_token);
                }
            }
        }];
    };
```

###本地项目管理
本地项目是指在手机或者平板上创建的项目，这些项目有可能是纯本地项目（还没有关联服务端项目），这些项目的`serverProjectId` 为空；有些项目关联了服务端项目，`serverProjectId` 为关联的服务端id
使用本地项目管理功能，需要先接入SDK

使用本地项目管理功能，需要用到
`com.realseei.android.viewmodel.LocalProjectManageVModel`

####获取项目列表
```
LocalProjectManageVModel.getAllProjects().forEach { projectModel ->
                    projectModel.projectName//项目名称
                    projectModel.serverProjectId//项目服务端id
                    projectModel.offLineId//项目本地id
                    projectModel.status//项目状态 0默认、1020拍摄中、1处理中、2处理失败、5处理成功
                    projectModel.device_type//采集类型：1手机拍、2全景相机、3云台拍、4伽罗华拍摄
                    projectModel.projectCreateTime//项目创建时间 2023-03-13 21:08:52
                    projectModel.titlePictureUrl//项目封面图
                    projectModel.is16kShoot//是否是高清拍摄
                }
                
```
>demo 中可参考 `com.realsee.android.capture.demo.main.ProjectListFragment#refreshData`

## SDK接入

为了顺利接入和正常使用，请确保接入环境如下：
-  `compileSdkVersion` < 31
-  `targetSdkVersion` <31
-  `kotlin` 环境
-  `AndroidX`支持

###`kotlin`环境搭建（已使用`kotlin`可忽略）

由于SDK和文档以及demo都是使用 	`kotlin` 语言开发，所以最好接入环境为 `kotlin` 语言开发环境
>以下接入过程只供参考   [官方接入文档](https://developer.android.com/kotlin/add-kotlin#groovy) 


1、在工程的 `build.gradle` 中添加如下

```groovy
dependencies {
        ...
        classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:1.6.10"
        ...
}
```
2、在所有模块的 `build.gradle` 中添加如下
```
//最上面添加
apply plugin: 'com.android.application'
apply plugin: 'kotlin-android'
//添加这个是为了使用一些扩展功能，如：直接访问 view 对象，不用 findViewById
apply plugin: 'kotlin-android-extensions'

...

dependencies {
    implementation 'androidx.core:core-ktx:1.2.0'
    implementation "org.jetbrains.kotlin:kotlin-stdlib:1.4.32"
}
```
>以上使用的 `kotlin` 相关版本最好跟文档中的保持一致

demo 实例位置：

```
        //初始化SDK
        RsOpenSdk.initSdk(this, object : SdkCallback() {
            override fun appId(): String {
                //后台申请的 APPID
                return "E0y67lyrQjyZRJr9"
            }

            override fun appSecret(): String {
                //后台申请的 APPSECRET
                return "BRQK3CUYOMZMIHU7DKI3LELXQ63MHTNC"
            }

            override fun debug(): Boolean {
                //是否开启debug
                return BuildConfig.DEBUG
            }

            override fun identity(): String {
                //如果不用区分用户身份，可以不重新这个方法
                return UserCenter.getIdentityId()
            }

            override fun userId(): String {
                //如果不用区分用户，可以不重新这个方法
                return UserCenter.getUid()
            }

            override fun host(): String {
                //设置所有的网络请求的host，本地化定制需求，无特殊需求，不要重新此方法
                return "http://app-gateway.realsee.com"
            }
            
             @WorkerThread
            override fun onCreateAndBindServiceProject(projectModel: ProjectModel): String {
                //拍摄sdk内部需要创建服务端项目时调用，返回创建好的服务端项目 id
                //会在子线程中调用
                VrProjectDataModel.createProject(
                    projectModel.projectName,

##项目本地预览
项目预览，是针对已经生成完成的项目，预览时，会先下载预览数据到本地，然后通过 `RsWebview` 进行本地预览

####初始化

使用项目本地预览功能需要初始化，可以在 `Application` 中初始化一次即可

demo 实例位置：
`com.realsee.android.capture.demo.App`

```
RsWebView.initCache(
            context,
            false,//是否在线预览，本地预览为 false
            RsWebView.CacheConfig().apply {
                //重新定义logo
                logoAssertName = "logo.png"
                //重新定义 poweredBy
                poweredByAssertName = "poweredBy.png"
                //内部会判断和保存这个版本号，如果大于已经保存的版本号，会重新设置缓存：eg，如果更换了loge 或者 poweredBy，一定要升级一下
                h5CacheVersion = 1
            },
            packageName,
            RsOpenSdk.webViewRequestInterceptor
        )

```

####下载预览数据
demo 实例位置：
`com.realsee.android.capture.demo.main.ProjectListFragment`

```
RsOpenSdk.downloadVrPreViewData(projectModel, { result ->
