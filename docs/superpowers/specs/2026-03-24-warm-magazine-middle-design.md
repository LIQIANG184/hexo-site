# 温润杂志型中间版设计说明

## 目标

在保留现有 Hexo + landscape 整体结构的前提下，把站点视觉从“默认老博客感”收拢到“温润、规整、安静的刊物感”。

本次调整定位为“温润杂志型中间版”：比纯轻量排版微调更有刊物气质，但明显轻于结构级的强杂志化改版。重点是视觉质量提升，而不是引入新的内容模块或新的内容组织方式。

## 范围

本次只处理以下四个区域：

1. `header`
2. 首页文章卡片
3. 文章正文排版
4. 侧栏视觉统一

列表页边界：

1. 本次“杂志条目”式卡片优化仅针对首页与首页分页列表
2. 分类页、归档页、标签页不作为本轮重点改造对象
3. 其他列表页如果沿用相同视觉语言，只做轻量一致性收口，不做与首页同强度的专门设计

## 明确不做

1. 不新增复杂模块
2. 不改首页信息架构
3. 不加入“精选头图区 / 特色文章区 / 分组卡片”等更强杂志化结构
4. 不调整文章内容组织方式
5. 不做重色块、强装饰、强悬浮的花哨设计

## 整体设计原则

### 1. 温润而克制

整体视觉应偏暖、柔和、安静，避免过于冷硬的边界，也避免明显的“UI 卡片堆叠感”。

### 2. 刊物感优先于组件感

优先通过留白、层级、排版节奏、轻边界和弱阴影建立秩序，而不是依赖厚重阴影或强色块去强调模块。

### 3. 保留现有结构

结构层不动，主要通过 `layout.css` 一层做样式收敛，尽量不引入模板层修改。

### 4. 统一视觉语言

首页卡片、正文块元素、侧栏小部件要共享同一套圆角、边界、阴影和间距语言，减少“每块像不同主题拼起来”的感觉。

## 方案概览

### Header

目标是让顶部首屏更像“刊头”，而不是默认 landscape 的旧式博客头图。

调整方向：

1. 强化站点标题作为第一视觉层级
2. 把副标题降到更轻的辅助层级
3. 让导航从装饰性存在变成清晰、规整的第三层级
4. 收窄过高的顶部区域，减少首屏被头图占满的感觉
5. 通过间距、字号、对齐关系优化层级，而不是靠新增装饰元素

预期效果：

1. 首屏更干净
2. 品牌感更清楚
3. 老主题气息被弱化，但整体仍是原站结构

### 首页文章卡片

目标是把首页文章列表做成“有完整轮廓，但不过分卡片化”的杂志条目。

确认方向：

1. 保留卡片轮廓
2. 边框和阴影收着做
3. 标题更突出
4. 摘要更稳
5. 日期和分类更规整

调整方向：

1. 统一卡片圆角、边界、阴影强度
2. 提升文章标题字号和标题区呼吸感
3. 收整 meta 信息的排列、间距和字重表现
4. 优化摘要行高和截断观感
5. hover 只提供轻微层次变化，不做明显悬浮

预期效果：

1. 首页更有秩序
2. 列表更像柔和纸页，而不是厚重组件列表
3. 视觉重心从杂项信息回到标题与摘要

### 正文阅读区

目标是让文章页更适合持续阅读，在不改内容结构的前提下提升阅读舒适度。

调整方向：

1. 细化正文宽度与行高节奏
2. 优化文章标题与各级标题之间的垂直间距
3. 统一段落、列表、图片、引用块、代码块的边界语言
4. 让图片圆角、引用块和代码块更协调
5. 保持中文阅读时的温和密度，避免过松或过挤

预期效果：

1. 阅读节奏更平稳
2. 块级元素之间不再割裂
3. 正文视觉温度更高，但不会偏离技术博客气质

### 侧栏

目标是让侧栏从“多个默认小部件堆叠”变成“更统一、更安静的辅助信息区”。

调整方向：

1. 优先突出分类、最近文章、皮肤切换
2. 弱化次要小部件的视觉存在感，但不改变侧栏信息架构
3. 统一 widget 的圆角、内边距、标题样式和边界表现
4. 减少噪音感，让侧栏成为正文的配套，而不是竞争视觉焦点

优先级定义：

1. 优先对象：分类、最近文章、皮肤切换
2. 可弱化对象：标签列表、标签云、归档，以及其他非核心浏览辅助型 widget
3. “突出皮肤切换”仅指可发现性和层级清晰度提升，不意味着做成强对比按钮或大面积强调块

边界说明：

1. “弱化”仅指样式层面的弱化，例如对比度、留白、边框存在感、标题强调度
2. 不包含删除模块、调整模块顺序、改模板结构
3. 当前已被首页脚本 `scripts/blog-index-layout.js` 隐藏的侧栏模块不在本次额外设计范围内
4. 如果仅靠 CSS 选择器无法稳定区分目标 widget，则优先退回到更保守的统一样式方案，不为了突出/弱化而引入脆弱选择器或模板改动

预期效果：

1. 侧栏更克制
2. 信息使用效率更高
3. 与正文、首页卡片形成统一语言

## 实现策略

### 文件层面

优先在 `source/css/layout.css` 中完成本次调整，继续与 `source/css/skin.css` 叠用。

### 改动策略

1. 尽量不触碰模板与内容生成逻辑
2. 优先复用现有变量与选择器结构
3. 新增样式以“局部增强”为主，不重写整套主题
4. 保持不同皮肤下的兼容性，特别是默认、深色、护眼三套皮肤

### 强度控制

本次改动的视觉强度要满足以下标准：

1. 比现在更精致、更统一
2. 比纯轻量版更有刊物气质
3. 但明显弱于“强杂志版”的结构级改造

### 响应式约束

1. 移动端仍沿用现有结构，不增加新的堆叠模块
2. `header` 的收窄在移动端要更谨慎，避免标题和导航拥挤
3. 首页卡片、正文块元素、侧栏 widget 在窄屏下优先保证呼吸感与可读性，而不是追求桌面端的装饰效果
4. 如果桌面端与移动端需要取舍，优先保证移动端阅读舒适度

## 实现锚点

以下不是硬性像素稿，而是实现时的控制区间，用来避免风格跑偏：

1. 圆角统一在“现有基础上略柔和”的区间，不做明显大圆角卡片化
2. 阴影以弱阴影为主，hover 只比静态状态轻微增强一档
3. `header` 调整以层级和留白为主，不新增装饰分隔物、徽章、插画
4. 首页标题层级要明显高于 meta，但不能大到形成封面式标题压迫感
5. 正文行高和段落间距以连续阅读舒适为先，避免过松导致信息发散
6. 引用块、代码块、图片圆角应统一到同一视觉语言，但仍保留技术内容的辨识度

## 验收标准

如果以下几点成立，说明本次设计达标：

1. 用户第一眼能感受到“更像刊物”，但不会觉得花或重
2. `header` 不再显得老气，首屏更清爽
3. 首页卡片更稳，标题和摘要层级更清晰
4. 正文更适合阅读，引用块和代码块不突兀
5. 侧栏更统一，不抢主内容
6. 站点结构和内容组织方式没有改变
7. 移动端没有因为这次收敛而出现拥挤、压缩或层级混乱

## 风险与边界

1. 如果阴影、边框或暖色处理过重，会偏向“装饰型博客”
2. 如果标题、副标题、导航层级拉得过开，可能削弱原主题识别度
3. 如果正文块元素统一过度，可能损失技术文章应有的信息区分度

因此本次实现必须坚持“收着做”的原则。

## 后续实现计划入口

在设计确认后，下一步进入实现规划，拆分为以下顺序：

1. `header` 层级与高度优化
2. 首页文章卡片细化
3. 正文排版与块元素统一
4. 侧栏小部件视觉统一
5. 深色 / 护眼皮肤兼容收尾
