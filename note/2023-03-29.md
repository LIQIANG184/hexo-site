#一 SDK接入
使用`podFile`导入SDK

1. 将`RSOpenSDK`文件夹拖到项目根目录
2. 打开`terminal`, cd到项目根目录
```
cd /Users/XXX
```
3. 初始化`pod`(如存在可跳过)
```
pod init
 ```
4. `Podfile`添加库
```
pod 'RSOpenSDK', :path => 'RSOpenSDK'
 ```
5. `pod` 更新
```
pod update
 ```
 
重新打开项目XXXX.xcworkspace,查看Pods目录,如下图是为正确引入
![image.png](https://upload-images.jianshu.io/upload_images/2791393-7c086aa5c4da51f9.png)

6. 开源库依赖说明:`AFNetworking`,`Masonry`,`Reachability`,`YYModel`


7. 其他: `隐私权限配置: Info.plist `蓝牙,摄像头

![image.png](https://upload-images.jianshu.io/upload_images/2791393-0e3300dacf0d0dda.png)

##2 设备管理
对象: `RSCapture.shareInstance.connectManager`, 详细见`demo``BSDeviceListViewModel.m`

```
/// 添加回调代理
- (void)addDelegate:(id<RSDeviceConnectDelegate>)delegate;
```
```
/// @param type 扫描的设备类型, 目前内部固定为 gimbal
/// @param timeout 扫描超时
/// @param identifier 传入 RSDeviceConnectDelegate 的 identifier, 用于区分超时回调
/// @note 扫描结果将会在 devicesDidScaned:scanType:error 中回调
/// @note timeout 为 0, 或者 identifier 为空, 都不会设置超时
/// @note 如果扫描到 wifi 设备, 说明这个wifi已经连上了, 会同时回调 扫描回调跟连接回调
- (void)scanDevice:(RSDeviceScanType)type timeout:(NSTimeInterval)timeout identifier:(NSString *_Nullable)identifier;
```
```
/// 停止扫描
- (void)stopScan;
```
```
/// 连接设备
/// @param device 待连接的设备
/// @param timeout 连接超时
/// @param identifier 传入 RSDeviceConnectDelegate 的 identifier, 用于区分超时回调
/// @note 连接结果将会在 connectDidchange:obj:error 中回调
/// @note timeout 为 0, 或者 identifier 为空, 都不会设置超时
- (void)connectDevice:(RSDeviceModel *)device timeout:(NSTimeInterval)timeout identifier:(NSString *_Nullable)identifier;
```
```
/// 取消当前连接
/// @note 如果已连接, 不会断开连接, 也不会有回调
- (void)cancelCurrentConnect;
```

```
/// 当前连接设备的电量, 0 ~ 1.0
@property (nonatomic, assign, readonly) CGFloat curDeviceBattery;
```
```
//获取历史设备
- (void)getAllHsitoryDevice:(void(^)(NSArray<RSDeviceModel *> *_Nullable list))completion;
```
```
/// 检查设备蓝牙开关以及权限
/// @note 调用这个方法不会触发 bluetoothStateDidChanged: 回调
- (void)checkBluetoothState:(void(^)(BOOL enabled, BOOL authorization))completion;
/// 判断当前是否有已连接的设备
- (BOOL)hasDeviceConnected;
```
